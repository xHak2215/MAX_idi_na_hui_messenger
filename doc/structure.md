# структура/архитектура сервера

## API
### общее

`/` (корень сервера) - при запросе отправляет статус код 200, создан для проверки соединения и пинга 

### медиа
работает преимущественно асинхронно или стримингом 

**пути по которым осуществляется взаимодействие:**

> заметка !
> разделить загрузку фото/видео и других файлов что бы оставить возможность отправлять их в исходном качестве 

- `/media/get` - отпровление файла к клиенту передача осуществляется стрименогм 
- `/media/uploadfiles` - загрузка файла на сервер передача осуществляется асинхроной отправкой пакетами по 1 КБ

### база данных
запросы к БД осуществляться через API, каталог: `/data_bese`

**пути по которым осуществляется взаимодействие:**

- `/data_bese/new_user` - создание нового пользователя
- `/data_bese/get_data_user` - получение данных о пользователе

### чаты

**пути по которым осуществляется взаимодействие:**
- `/chat/chat_message` - получение содержания чата
- `/chat/send_message` - отправка сообщения
- `/chat/creat_chat` - создание чата
- `/chat/get_chat` - получение информации о чате
- `/chat/get_message` - получение сообщения в чате по его ID

**структура ответов:**

ответы приходят в json формате

пример ответа от БД:

`{'is_ok': True, 'error_code': None, 'detalis': None, 'data': {'chat_id': 1, 'name': 'чат 1', 'avatar': 0, 'about': 'test chat', 'users_list': ['user228'], 'private': True}}`

- `is_ok` - показывает есть ли ошибка, True есть False нет 
- `detalis` - детали ошибки для лучшего понимания и дебага, None если ошибки не было
- `error_code` - это статус код ошибки, если ошибки нет равен None 
- `data` - основные данные 

<details><summary>коды ошибок</summary><p> 

- `404 Not Found` — самая распространённая ошибка при пользовании Интернетом, основная причина — ошибка в написании адреса Web-страницы. Сервер понял запрос, но не нашёл соответствующего ресурса по указанному URL.  

- `403 Forbidden` — сервер понял запрос, но он отказывается его выполнять из-за ограничений в доступе для клиента к указанному ресурсу. Иными словами, клиент не уполномочен совершать операции с запрошенным ресурсом.

- `500 Internal Server Error` — любая внутренняя ошибка сервера, которая не входит в рамки остальных ошибок класса.
</p></details>

### токены:
для индикации пользователя и зашиты от банального перебора пароля используются уникальные токены.
пример токена:`2cf5189a992695fb402f0bce3724dd2b139ba9f22c60d0a6fd817c993beede40`

используется в получении чатов и личной информации, храниться в базе данных и авторизованном устройстве, создаться и отправляться при регистрации клиенту

## хранение медиа 

хранение осуществляется в директории `/media`

**файлы:**

при загрузке файлов каждому выдаться уникальный ID который являться как индикаторам так и порядковым номером 

пример: <a href="/media/0.jpg">0.jpg</a> 
 
так же изображения и видео будут сжиматься на 20%  

**чаты:**

чаты хроняться как отдельные файлы в формате json, дериктория хранения: `/chats` пример имени файла: `chat_0.json`

0 - это ID чата 

**данные хранимые сообщением:**

```json
{
"2":{"user":"юзер отпрорвитель (login)", "message":"hello, world!", "time":1764180929.405108, "reply_to":-1}
}
```

`"2"` - ключ вызова и ID сообщения

`user` - логин пользователя

`message` - текст сообщения

`time` - время отправки 

`reply_to` - ID сообщения на тоторое был совершен ответ, если вообщение ни на что не отвечает то -1
